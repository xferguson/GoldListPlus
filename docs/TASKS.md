# Tasks — Gold List Plus

> **Maintained by the tech-lead subagent.** Each task is a single TDD cycle: QA writes the failing tests for the acceptance criteria, the Implementer makes them pass, the Code Reviewer audits the diff.
>
> Conventions:
> - Tasks are completed in order unless `Depends on:` says otherwise.
> - Files listed under `Files touched:` are the expected surface. New files outside that list need Tech Lead sign-off.
> - AC are numbered. Tests cite the task ID and AC number (e.g. `TASK-003 AC-2`).

### TASK-004: Pure distillation logic
- [ ] Status
- **Files touched:** `src/lib/distillation.ts`, `src/lib/distillation.test.ts`, `src/lib/tiers.ts`.
- **Depends on:** none (purely typed; data model copy lives here as type imports).
- **Acceptance criteria:**
  1. `nextTier('bronze') === 'silver'`, `nextTier('silver') === 'gold'`, `nextTier('gold') === null`.
  2. `flagCardForDistillation(rating, bookSettings)` is true for `wrong` regardless of any setting.
  3. `flagCardForDistillation` matrix across (`wrong`,`hard`,`moderate`,`easy`) × all 8 combinations of the three `autoDropOn*` flags is exhaustively covered with one test each.
  4. `flagsForPage(page, cards, latestEventByCardId, settings)` returns a `Map<string, boolean>` keyed by cardId with correct flags; Cards with no ReviewEvent (shouldn't happen but defend it) are not flagged.
  5. `finalizePage({ parent, builderEntries, now, intervalDays })` returns a plan:
     - `childPage.tier = nextTier(parent.tier)` and `childPage.reviewableAt = nextTier === 'gold' ? null : now + intervalDays * 86_400_000`.
     - `childPage.parentPageId === parent.id`.
     - `newCards` has one Card per builder entry; each has `parentIds` from the entry; `pageId === childPage.id`; `bookId === parent.bookId`.
     - `archivedCardIds` contains every `cardId` from `parent.cardIds`.
  6. `finalizePage` throws if `parent.tier === 'gold'`.
- **Out of scope:** persistence; the plan is in-memory only.

### TASK-005: Repos — books, pages, cards, reviews
- [ ] Status
- **Files touched:** `src/db/repos/{books,pages,cards,reviews}.ts`, `src/db/repos/*.test.ts`.
- **Depends on:** TASK-003, TASK-004.
- **Acceptance criteria:**
  1. `books.create`, `books.get`, `books.update`, `books.list`, `books.remove` (cascades to pages, cards, reviews).
  2. `pages.create`, `pages.get`, `pages.listByBook(bookId)`, `pages.listDue(now)` returns only `reviewableAt !== null && reviewableAt <= now && reviewedAt == null`.
  3. `pages.finalize(plan)` writes the new Page, new Cards, archives parent Cards (sets `archivedAt`), and links parent.`childPageId` / parent.`reviewedAt` — all inside one Dexie transaction. Failures roll back.
  4. `cards.create`, `cards.get`, `cards.listByPage`, `cards.update`, `cards.remove`. Update/remove fail on a Card whose page is `reviewedAt`-set (locked).
  5. `reviews.append(event)`, `reviews.listByCard(cardId)`, `reviews.listByPage(pageId)`, `reviews.latestPerCardForPage(pageId)` returns a `Map<string, ReviewEvent>`.
  6. All tests run against `fake-indexeddb`.
